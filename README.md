# Desafio DIO - Engenharia de Prompts (Análise de Feedback)

Este repositório contém a entrega do Desafio Criativo de Engenharia de Prompts. Abaixo está o comando consolidado e refinado para análise automatizada de feedbacks de Clientes Bancários.
---

### 🚀 Prompt Final Estruturado

```text
Atue como analista de atendimento e qualidade.

Sua tarefa é analisar os comentários da Google Play e App Store sobre o aplicativo do banco para identificar por que as pessoas estão reclamando tanto do Pix depois da última atualização.

Contexto: O time de desenvolvimento e o gerente de produto vão usar esse relatório para descobrir quais telas estão quebrando e corrigir os erros mais críticos ainda esta semana.

Dados disponíveis: Uma lista em texto contendo a data da postagem, a nota de 1 a 5 e o desabafo do cliente.

Instruções de análise:
- Classifique os feedbacks por tipo de problema (como saldo sumido, erro no teclado ou demora) e por gravidade.
- Identifique os principais padrões, problemas, elogios e oportunidades.
- Aponte evidências nos dados fornecidos.
- Sugira ações práticas para o time técnico e para a equipe que desenha as telas do app.

Formato da resposta: Uma tabela simples com as colunas: [Problema | O que o cliente disse | O que precisamos ajustar].

Restrições:
- Use apenas os dados fornecidos.
- Não invente números, causas ou conclusões.
- Não exponha dados pessoais ou sensíveis.
- Informe limitações quando os dados não forem suficientes.
- Use linguagem totalmente direta, simples e sem enrolação.
